# 작업환경 구성 보고서

## 목적

시뮬레이션 담당자가 바로 작업을 시작할 수 있도록 로컬 작업공간의 번호 폴더, 규칙 문서, 원본 데이터셋, 최종 모델, 시뮬레이션 입력 계약을 정리했다.

## 작업 내용

- 로컬 기준 번호 폴더를 정리했다.
- `02_전략_기획`을 앞으로 시뮬레이션 방향성과 계획을 저장하는 폴더로 정의했다.
- 큰 작업 단위 보고서를 누적할 `06_보고서` 폴더를 만들었다.
- 기준 HeatGridAgent 저장소에서 원본 PreDist 데이터셋과 최종 모델 handoff 패키지를 가져왔다.
- 시뮬레이션 입력 JSON 계약과 예시를 `05_시뮬레이션`에 배치했다.

## 현재 폴더 구조

| 경로 | 역할 |
|---|---|
| `00_작업규칙` | Git, 보고서, 작업 체크리스트 |
| `01_프로젝트개요` | 기준 프로젝트 개요와 문제 정의 |
| `02_전략_기획` | 시뮬레이션 방향성과 실행 계획 |
| `03_데이터셋` | 원본 PreDist 데이터셋과 metadata |
| `04_최종모델` | 완료 모델 handoff 패키지와 관련 문서 |
| `05_시뮬레이션` | 입력 계약, 예시, 화면/데모 작업 |
| `06_보고서` | 큰 작업 단위 결과 보고서 |

## 입력 자료

- `01_프로젝트개요/01_HeatGridAgent_프로젝트개요.md`
- `02_전략_기획/01_HeatGridAgent_디벨롭_메모.md`
- `02_전략_기획/02_HeatGridAgent_BMS_통합_검토.md`
- `03_데이터셋/PreDist/predist_dataset.zip`
- `04_최종모델/m1_specialist_handoff.zip`
- `04_최종모델/m1_specialist_package.zip`
- `05_시뮬레이션/contracts/simulation_input.schema.json`
- `05_시뮬레이션/examples/simulation_input.example.json`

## 검증

- 시뮬레이션 JSON 스키마와 예시 JSON 파싱을 확인했다.
- `pyproject.toml` 파싱을 확인했다.
- 원본 데이터 zip과 최종 모델 zip이 `.gitignore`로 제외되는지 확인했다.
- 예전 번호 폴더 참조가 남아 있는지 검색했다.

## 남은 작업

- 최종 모델 handoff 패키지 내부 구조를 확인해 실제 자동감지 출력 형태를 확정한다.
- 원본 데이터셋에서 대표 기계실 1건을 고른다.
- RAG/연결/DB 담당 산출물을 받으면 `05_시뮬레이션/inputs/team_outputs`에 맞춰 연결한다.
