# 보고서

큰 작업을 하나 끝낼 때마다 결과 보고서를 이 폴더에 누적한다.

## 파일명 규칙

- 번호와 주제를 함께 쓴다.
- 예: `01_작업환경_구성_보고서.md`
- 예: `02_대표시나리오_JSON_연결_보고서.md`

## 기본 구성

1. 목적
2. 변경/작업 내용
3. 사용한 입력 자료
4. 검증 결과
5. 남은 한계와 다음 작업

## 작성 기준

- 한국어 Markdown으로 작성한다.
- 수치나 파일명은 실제 확인한 경로를 적는다.
- 원본 데이터 기준, 모델 산출물 기준, 화면 예시 기준을 섞지 않는다.
- 검증하지 못한 내용은 숨기지 않고 적는다.
