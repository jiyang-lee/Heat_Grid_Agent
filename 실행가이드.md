# HeatGrid 로컬 실행 가이드

## 빠른 실행

PowerShell에서 저장소 루트로 이동한 뒤 실행합니다.

```powershell
.\start-local.ps1
```

PostgreSQL/pgvector와 FastAPI를 Docker로 시작하고, 프론트엔드를 LAN 접속 가능하도록 별도 PowerShell 창에서 시작합니다.

처음 실행하면 루트의 `.env.example`에서 `.env`를 만들고, LLM을 사용할 팀원은 자신의 `OPENAI_API_KEY`를 `.env`에 입력합니다. 실제 키는 GitHub에 올리지 않습니다.

| 대상 | 주소 |
|---|---|
| 내 PC | `http://127.0.0.1:5173` |
| 같은 네트워크 사용자 | `http://내_PC의_IP:5173` |
| 백엔드 상태 | `http://127.0.0.1:8003/health` |

실행 스크립트가 내 PC의 IP를 출력합니다. 다른 PC에서 접속하지 못하면 Windows 방화벽에서 TCP 5173 인바운드 규칙을 허용하고 두 PC가 같은 네트워크인지 확인하세요.
관리자 PowerShell로 실행하면 이 규칙을 자동으로 추가합니다.

## 팀원 최초 설정

```powershell
git pull origin develop2
Copy-Item .\env.example .\.env
```

1. Docker Desktop을 실행한다.
2. `.env`의 `OPENAI_API_KEY`에 본인 키를 입력한다. 키가 없으면 화면과 DB 기능은 실행되지만 LLM 생성 기능은 사용할 수 없다.
3. 다른 PC에서도 프론트를 열어야 하면 `start-local.ps1`을 관리자 PowerShell로 한 번 실행해 TCP 5173 방화벽 규칙을 추가한다.

### ZIP 데이터까지 동일하게 복원

공유받은 `HeatGrid_Local_Environment_*.zip`을 풀어서 나오는 두 파일을 아래 위치에 둔다. 대용량·개인 로컬 데이터이므로 GitHub에는 올리지 않는다(최초 1회만 수동으로 준비하면 되고, 이후에는 `git pull`만으로 코드·Docker가 최신화된다).

- `database_data.dump` → 저장소 루트
- `demo_replay.zip` → `data\replay\demo_replay.zip`

그 다음 실행한다. `start-local.ps1`이 `data\replay\`에 있는 zip을 감지해 재생 데이터셋을 자동으로 임포트하므로, 데이터셋이 아직 없을 때만 별도로 `import_replay_dataset.ps1`을 실행하면 된다.

```powershell
.\restore_database.ps1
.\start-local.ps1
```

## 수동 실행

```powershell
docker compose up -d --build --wait
Copy-Item .\frontend\.env.example .\frontend\.env -ErrorAction SilentlyContinue
Set-Location .\frontend
npm ci
npm run dev -- --host 0.0.0.0 --port 5173
```

## 선택: ZIP의 DB 데이터 복원

`database_data.dump`를 받은 경우에만 실행합니다. 기존 로컬 PostgreSQL 업무 데이터를 초기화합니다.

```powershell
.\restore_database.ps1
```

## 종료

```powershell
docker compose down
```

`docker compose down -v`는 DB 볼륨까지 삭제합니다.
