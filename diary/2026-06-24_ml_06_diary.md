## 고정된 모델 구조

프로젝트 ML 구조는 다음으로 고정한다.

```text
Isolation Forest -> anomaly_score
LightGBM -> risk_probability / risk_score
```

해석 규칙:

- Isolation Forest 결과는 “고장”이 아니라 정상 패턴 대비 이상징후다.
- LightGBM 결과는 “고장 확정”이 아니라 `faults.csv`의 고장신고 전 위험 패턴과의 유사도다.
- `faults.csv`의 시점은 실제 고장 발생 시점이 아니라 신고 시점이다.
- 최종 결과는 Priority Engine과 Agent가 사용할 위험 정보다.

## 잘못된 개선 해석 수정

한때 03 normal reference filter로 validation/holdout normal을 평가 집합에서 제거한 결과가 개선처럼 보였다.

이 해석은 폐기했다.

이유:

- 평가 집합 자체가 줄어들었기 때문이다.
- validation/holdout은 원래 분포를 유지해야 한다.
- normal reference filter는 학습 사용 여부 판단에는 쓸 수 있지만, 평가 row를 제거하면 성능 비교가 왜곡된다.

최종 방식:

- 04는 평가 normal row를 유지한 `trainable_windows.csv`를 만든다.
- 05는 전체 trainable window에 anomaly score를 산출한다.
- 06은 `ml_window_dataset.csv`의 control column을 merge하되, 모델 feature로 쓰지 않는다.
- validation/holdout 평가 row는 제거하지 않는다.

- 단순 모델 튜닝으로 해결할 문제가 아니다.

- 06 다음 단계는 07이 아니라 label/normal/time 정의 보강이다.
