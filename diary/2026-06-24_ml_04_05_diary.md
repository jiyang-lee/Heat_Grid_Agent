## 04. feature selection

파일:

```text
PREPROCESSING/osj/04_feature_selection.ipynb
PREPROCESSING/docs/04_feature_selection.md
```

역할:

- 03번의 `ml_window_dataset.csv`에서 실제 학습에 사용할 행과 feature 컬럼을 확정한다.
- metadata 컬럼과 모델 입력 feature 컬럼을 분리한다.
- 결측률, 상수 컬럼, 제조사별 coverage 기준으로 baseline feature를 선택한다.
- strict / relaxed 학습 후보셋을 분리한다.
- train split 기준 결측 대체값을 저장한다.

중요 기준:

```text
feature 선택 통계 기준:
strict 후보셋 + split_time_based == train
```

기본 학습 후보:

```text
use_for_supervised_training == True
label in {normal, pre_fault}
data_quality_issue == False
```

feature 선택 기준:

```text
numeric 또는 bool dtype
strict train split 기준 결측률 <= 50%
strict train split에서 상수 컬럼 제외
양 제조사 모두에서 non-null 값 존재
```

결측 대체 기준:

```text
numeric feature: strict train split median
bool feature: strict train split mode
```

주요 산출물:

```text
data/processed/ml_features/trainable_windows.csv
data/processed/ml_features/trainable_windows_strict.csv
data/processed/ml_features/trainable_windows_relaxed.csv
data/processed/ml_features/trainable_windows_strict_imputed.csv
data/processed/ml_features/trainable_windows_relaxed_imputed.csv
data/processed/ml_features/feature_columns.csv
data/processed/ml_features/metadata_columns.csv
data/processed/ml_features/imputation_values.csv
```

현재 검증 결과:

```text

```

## 05. baseline anomaly model

파일:

```text
PREPROCESSING/osj/05_baseline_anomaly_model.ipynb
PREPROCESSING/docs/05_baseline_anomaly_model.md
```

역할:

- 04번의 `trainable_windows.csv`를 사용해 baseline 이상탐지 모델을 학습한다.
- 정상 train 구간만 사용해 normal behaviour 기준 모델을 만든다.
- 각 window별 anomaly score와 threshold 기반 label을 저장한다.
- split별 기본 성능과 threshold sweep 결과를 저장한다.
- 모델과 scaler를 재사용 가능한 파일로 저장한다.

학습 기준:

```text
split_time_based == train
label == normal
```

사용 모델:

```text
IsolationForest anomaly baseline
```

프로젝트 원래 계획 기준으로는 05번의 핵심 모델을 Isolation Forest로 둔다.
최종 모델 흐름은 `Isolation Forest + LightGBM` 2단계 구조다.

평가 기준:

```text
split_time_based
split_substation_based
```

threshold 기준:

```text
train normal score quantile
0.95 / 0.975 / 0.99
```

주요 산출물:


## 주의할 점

### 1. 00번은 자동 전체 실행에서 제외하는 것이 좋다

00번은 외부 다운로드가 포함된다.
이미 raw data가 존재하는 상태에서는 매번 실행하지 않아도 된다.

### 2. 05번은 아직 baseline이다

05번 결과는 최종 위험도 모델이 아니다.
현재 결과는 정상 패턴 대비 이상점수를 만드는 1차 baseline이다.

Agent에게 넘길 최종 `risk_score`, `severity`, `main_abnormal_sensors`는 06~07 단계에서 더 정리해야 한다.

### 3. event 단위 holdout은 아직 별도 구현되지 않았다

기존 기록에 따르면 time split에서 fault event 1개가 갈라지는 케이스가 있었다.

```text
manufacturer 1 / substation 28 / fault_event_id 47
```

06번 이후 평가 단계에서 event 단위 leakage 검사를 추가하는 것이 좋다.
