# 시뮬레이션 방향성 계획

## 목적

자동감지 에이전트 결과, 원본 PreDist 데이터셋, RAG 답변, 연결/DB 담당 산출물을 한 화면에서 이어 보이게 만들어 운영 흐름을 검증한다.

## 기본 방향

1. 원본 데이터셋에서 대표 기계실과 시간 구간을 고른다.
2. 완료된 최종 모델 또는 자동감지 에이전트 출력으로 위험 라벨과 점수를 만든다.
3. RAG 담당 산출물로 원인 설명, 근거, 점검 포인트를 붙인다.
4. 연결/DB 담당 산출물로 현장, 설비, 센서 키를 연결한다.
5. `05_시뮬레이션`에서 JSON 입력 하나로 화면 또는 실행 결과를 확인한다.

## 우선순위

1. 대표 사례 1건을 끝까지 연결한다.
2. 입력 JSON 계약을 실제 모델/데이터 산출물에 맞게 보정한다.
3. 화면에서 우선순위, 감지 결과, RAG 근거, 점검 포인트가 한 번에 읽히게 만든다.
4. 같은 구조로 다른 기계실 사례도 교체 가능하게 만든다.

## 현재 가정

- 원본 데이터셋은 `03_데이터셋/PreDist`를 사용한다.
- 최종 모델 handoff 패키지는 `04_최종모델`을 사용한다.
- 팀 산출물이 아직 없으면 `05_시뮬레이션/examples/simulation_input.example.json`으로 임시 연결한다.

## 산출물 관리

- 실행 결과와 화면 작업: `05_시뮬레이션`
- 큰 작업 단위 보고서: `06_보고서`
