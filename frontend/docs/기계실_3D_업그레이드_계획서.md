# 기계실 관제 화면 3D 렌더 느낌 업그레이드 — 계획서

## 1. 배경 / 목적
기계실 관제(`기계실 관제 · MACHINE ROOM`) 화면은 100% 코드로 그린 평면적인 SVG 아이소메트릭 스키매틱(설비 박스 + 배관)이었다. 참고 이미지(다크 3D 아이소메트릭 플랜트)처럼 **"기계가 돌아가는" 3D 렌더 느낌**을 주는 것이 목표.

## 2. 확정 방향
- **코드 기반 3D 업그레이드**: 외부 렌더 이미지 에셋 없이 기존 SVG를 강화한다. → 31개 단지 데이터 연동·다크 테마 유지, 전체 자체 구현 가능.
- **설비 기계만 애니메이션**(펌프 임펠러 회전, 제어반 표시등 점멸 등). **배관은 정적**.
- **금속 재질 + 그림자/바닥 반사** 추가 (3D 렌더 체감의 핵심).
- **범위 제외**: 플로팅 상태 라벨 카드+리더선, 방 배경(벽/천장/조명), 고장 예측 콜아웃 배지, 배관 흐름 애니메이션, three.js 실시간 3D.

## 3. 제약
- **`frontend/` 폴더 내에서만** 작업. `/api` 계약 표면 무관 — 순수 프론트 시각/도메인 계층.
- `RoomSchematic` ↔ `DetailAside` 동기화 계약(props: `complex`, `selMachine`, `onSelectMachine`) 유지.
- 프로젝션(`iso`, `boxGeom`), viewBox `0 0 900 560`, `translate(360,140)`, 클릭/칩/이름 라벨, 그리기 순서(painter's algorithm) 유지.

## 4. 설계

### 4.1 애니메이션 종류를 데이터로 구동 — `src/domain/machines.ts`
- `Machine`에 `kind: 'pump' | 'hex' | 'tank' | 'valve' | 'panel'` 추가.
- 매핑: `hex→hex`, `pump1/pump2/makeup→pump`, `exp→tank`, `prv→valve`, `ctrl→panel`.

### 4.2 렌더 업그레이드 — `src/room/RoomSchematic.tsx` (주 작업)
- **`<defs>`**: `#spec`/`#specTop`(흰색→투명 스펙큘러, 상태색 공용), `#softShadow`(feGaussianBlur).
- **접지 그림자**: 설비별 `cbase` 아래 블러 타원.
- **바닥 반사**: 박스를 base 기준 세로 미러 + 저투명 복제.
- **금속 재질**: top/right 면 위에 `url(#spec)` 하이라이트 오버레이.
- **배관**: 정적 유지, 긴급 설비 연결 배관만 빨간 톤 강조.

### 4.3 설비 심볼/애니메이션 (감시되는 설비만)
- **pump**: 회전 임펠러(`@keyframes spin`).
- **panel**: LED 점멸(`@keyframes ledBlink`, delay 엇갈림).
- **hex**: 히트 글로우 밴드(`@keyframes heatBreathe`).
- **tank/valve**: 정적(밸브는 핸드휠 심볼).
- 긴급=기존 `.pulse`, 선택=기존 `.sel` 유지. 회색(센서 미탑재)=정적.

### 4.4 스타일 — `src/theme.css`
- keyframes `spin`/`ledBlink`/`heatBreathe`, 클래스 `.impeller`/`.led`/`.heatband`(`transform-box: fill-box`).
- `@media (prefers-reduced-motion: reduce)`로 애니메이션 정지.

## 5. 변경 파일
- `src/room/RoomSchematic.tsx` (주)
- `src/domain/machines.ts`
- `src/theme.css`

## 6. 검증 방법
1. `npx tsc -b --noEmit` → 타입 오류 0.
2. 지도 관제 → 단지 클릭 → 기계실 진입.
3. DOM 검증: 펌프 `.impeller`, 제어반 `.led`, 열교환기 `.heatband`(감시 시), 긴급 `.pulse`, 선택 `.sel`, `defs`/접지 그림자 존재. 센서 미탑재 설비는 애니메이션 클래스 없음.
4. 설비 클릭 ↔ `DetailAside` 선택 동기화.
5. 콘솔 오류 0, vite 오버레이 없음.
