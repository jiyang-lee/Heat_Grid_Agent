# HeatGridAgent 시뮬레이션 개요

## 담당 범위

시뮬레이션 담당은 원본 데이터셋, 완료된 자동감지 에이전트 출력, RAG 답변, 연결/DB 담당 산출물을 받아 운영자가 확인할 수 있는 화면 흐름으로 묶는다.

## 현재 입력 후보

- 원본 PreDist 데이터셋
- 완료된 자동감지 에이전트 출력
- 우선순위 카드
- RAG Output
- Priority-RAG 연결 기준 확인 결과
- DB/JSON 스키마 매핑 결과

## 첫 MVP 목표

대표 사례 1건을 고정하고, `05_시뮬레이션/examples/simulation_input.example.json` 형태의 입력 JSON을 화면에서 읽어 다음 정보를 한 번에 보여준다.

- 우선순위 등급
- 모델 감지 라벨과 위험 점수
- RAG 설명과 근거
- 현장/설비/센서 매핑
- 권장 점검 포인트

## 경계

- 모델 학습과 추론 로직은 자동감지 담당 범위다.
- RAG 생성 로직은 RAG 담당 범위다.
- 스키마 확정과 DB 매핑은 DB/스키마 담당 범위다.
- 시뮬레이션 담당은 받은 결과를 화면과 데모 흐름으로 연결한다.
