# HeatGridAgent Simulation

담당: 이지양

## 역할

자동감지 에이전트가 만든 우선순위 카드, RAG 답변, 연결/DB 담당 산출물을 하나의 JSON 입력으로 받아 눈으로 확인 가능한 시뮬레이션 화면을 만든다.

이 저장소는 `C:\3rd_Project\HeatGridAgent`에서 시뮬레이션 담당에게 필요한 자료만 가져온 작업 공간이다.

## 현재 들고올 수 있는 것

- 원본 PreDist 데이터셋
- 완료된 모델 또는 자동감지 에이전트 출력
- 우선순위 카드 산출물

## 받아야 하는 입력

- `priority_card`: 자동감지 에이전트가 만든 출동/점검 우선순위
- `model_detection`: 원본 데이터셋 또는 모델 추론에서 나온 이상 감지 결과
- `rag_answer`: RAG 담당의 원인 설명, 근거, 권장 점검 항목
- `connection_check`: 연결 담당의 Priority-RAG 연결 기준 확인 결과
- `db_mapping`: DB/스키마 담당의 현장, 설비, 센서 매핑 결과

입력 규격은 `05_시뮬레이션/contracts/simulation_input.schema.json`, 예시는 `05_시뮬레이션/examples/simulation_input.example.json`을 기준으로 맞춘다.

## 출력

- 대표 사례 1건 이상을 보여주는 시뮬레이션 화면
- 화면에서 확인 가능한 정보:
  - 우선순위 등급
  - 자동감지 결과와 모델 점수
  - RAG 설명과 근거 문서
  - 현장/설비/센서 매핑
  - 권장 점검 포인트

## 작업 경계

- 모델 학습과 추론 로직 자체는 자동감지 에이전트 담당 범위다.
- RAG 답변 생성은 RAG 담당 범위다.
- Priority-RAG 연결 기준 검증은 연결 담당 범위다.
- DB 스키마 정의와 원천 매핑 확정은 DB/스키마 담당 범위다.
- 시뮬레이션 담당은 위 산출물을 조합해 화면에 보이는 흐름과 데모 데이터를 완성한다.

- 보고서 규칙 요약: `00_작업규칙/02_보고서_규칙.md`
- Codex 작업 규칙: `AGENTS.md`

## 구현 순서

1. `01_프로젝트개요`와 `02_전략_기획`을 기준으로 시뮬레이션 목표와 범위 확정
2. `03_데이터셋/PreDist`와 `04_최종모델`의 실제 입력 자료 확인
3. `05_시뮬레이션/examples/simulation_input.example.json`로 화면에 필요한 필드 확정
4. 우선순위, 감지 결과, RAG 설명, DB 매핑을 한 화면에서 읽히는 구조로 배치
5. 원본 데이터셋의 대표 기계실 1건을 실제 시나리오로 고정
6. 완료된 자동감지 에이전트 출력으로 예시 JSON 교체
7. RAG/연결/DB 담당 산출물 수령 후 같은 JSON 구조에 끼워 넣기
8. 큰 작업을 끝낼 때마다 `06_보고서`에 보고서 누적

## 완료 기준

- 대표 사례 1건 이상을 화면 또는 실행 결과로 확인할 수 있어야 한다.
- 우선순위, 근거 설명, 점검 포인트가 데모에서 이어져 보여야 한다.
- 입력 JSON만 바꾸면 다른 기계실 사례도 같은 화면 구조로 확인 가능해야 한다.
