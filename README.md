# agent/connect

담당: 오승주

## 역할

사용한 DB 데이터와 운영 현장 정보를 연결하는 매핑 테이블 또는 문서를 만든다.

## 입력

- 사용한 데이터(DB)
- 설비, 수요처, 단지, 책임구간 관련 보조 자료
- `agent/schema`의 스키마 정의

## 출력

- `substation_id`와 실제 설비/주소/단지/책임구간을 연결한 매핑 테이블
- RAG와 시뮬레이션에서 참조할 수 있는 연결 문서

## 작업 경계

- JSON/SQL 스키마 자체는 `agent/schema`에서 담당한다.
- LLM 답변 생성은 `agent/rag`에서 담당한다.
- 데모 화면 구현은 `agent/simulation`에서 담당한다.

## 완료 기준

- 대표 `substation_id`가 실제 현장명, 주소, 단지 코드, 책임구간과 연결되어야 한다.
- 매핑 결과는 RAG 입력으로 바로 사용할 수 있는 표 또는 문서 형태여야 한다.
