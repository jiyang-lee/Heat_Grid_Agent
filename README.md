# agent/simulation

담당: 이지양

## 역할

RAG 구축 결과와 완성된 앞단 데이터를 이용해 눈으로 확인 가능한 데모를 만든다.

## 입력

- `agent/rag`의 RAG 구축 결과와 LLM 답변
- `agent/connect`의 매핑 테이블 또는 문서
- `agent/schema`의 JSON/DB 스키마
- 완성된 앞단 데이터

## 출력

- 운영 흐름을 확인할 수 있는 데모
- 필요 시 서버/API 연결 또는 화면 확인용 문서

## 작업 경계

- 스키마 정의는 `agent/schema`에서 담당한다.
- 현장 매핑은 `agent/connect`에서 담당한다.
- RAG 답변 생성은 `agent/rag`에서 담당한다.

## 완료 기준

- 대표 사례 1건 이상을 화면 또는 실행 결과로 확인할 수 있어야 한다.
- 우선순위, 근거 설명, 점검 포인트가 데모에서 이어져 보여야 한다.
