# 보고서 규칙

이 저장소의 보고서는 `C:\3rd_Project\HeatGridAgent`의 산출물 패턴을 따른다.

## 저장 위치

- 큰 작업 단위 보고서: `06_보고서`
- 시뮬레이션 화면/실행 중간 산출물: `05_시뮬레이션/outputs`
- 작업 체크리스트와 규칙 문서: `00_작업규칙`

## 파일명

- 번호와 주제를 함께 쓴다.
- 예: `01_작업환경_구성_보고서.md`
- 예: `02_우선순위_RAG_연결_검증_보고서.md`

## 기본 구성

1. 목적
2. 입력 자료
3. 처리 기준
4. 결과
5. 검증
6. 남은 한계

## 작성 기준

- 한국어 Markdown을 기본으로 한다.
- 수치와 판단은 출처 파일을 함께 적는다.
- 원본 파일 기준, 전처리 결과 기준, 화면 예시 기준을 섞지 않는다.
- 표와 짧은 문단을 우선 사용한다.
- 검증하지 않은 내용은 `미검증`, `가정`, `추후 확인`으로 표시한다.
- 노트북이나 스크립트로 만든 결과는 실행 파일, 입력 파일, 출력 파일을 함께 적는다.
