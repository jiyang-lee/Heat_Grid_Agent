# AGENTS.md

이 저장소는 HeatGridAgent 시뮬레이션 담당 작업 공간이다. `C:\3rd_Project\HeatGridAgent`에서 필요한 프로젝트 개요, 전략 기획, 원본 데이터셋, 최종 모델만 가져와 사용한다.

## 작업 원칙

- 사용자의 설명을 사실로 가정하지 말고 실제 파일, 코드, 로그, 실행 결과를 먼저 확인한다.
- 작업은 현황 파악, 구현, 테스트, 결과 보고 순서로 끝낸다.
- 관련 없는 리팩터링, 설정 변경, 파일 이동은 하지 않는다.
- 원본 데이터셋과 모델 산출물은 사용자가 명시하지 않으면 복사하거나 수정하지 않는다.
- 루트에는 임시 파일을 만들지 않는다. 목적에 맞는 번호 폴더 아래에 둔다.

## 폴더 규칙

- 작업 규칙과 당일 TODO는 `00_작업규칙`에 둔다.
- 프로젝트 개요와 문제 정의는 `01_프로젝트개요`에 둔다.
- 시뮬레이션 방향성과 실행 계획은 `02_전략_기획`에 둔다.
- 원본 PreDist 데이터셋과 metadata는 `03_데이터셋/PreDist`에 둔다.
- 완료 모델 handoff 패키지는 `04_최종모델`에 둔다.
- 시뮬레이션 개요, 입력 계약, 예시, 화면 MVP는 `05_시뮬레이션`에 둔다.
- RAG/연결/DB 담당 추가 산출물은 `05_시뮬레이션/inputs/team_outputs`에 둔다.
- 화면이나 실행 결과는 `05_시뮬레이션/outputs`에 둔다.
- 큰 작업 단위 결과 보고서는 `06_보고서`에 둔다.

## Git 규칙

- 커밋 메시지는 한국어 Conventional Commit 형식을 따른다.
- 예: `feat: 시뮬레이션 입력 JSON 계약 추가`
- 예: `docs: 시뮬레이션 보고서 규칙 정리`
- 대용량 원본 데이터 zip, 모델 zip, `.env`, 가상환경, 캐시는 Git에 올리지 않는다.
- 원격 푸시와 배포는 사용자 요청 없이 하지 않는다.

## 보고서 규칙

- 보고서는 한국어 Markdown을 기본으로 작성한다.
- 보고서 파일명은 번호와 주제를 붙인다. 예: `01_작업환경_구성_보고서.md`
- 보고서에는 최소한 목적, 입력 자료, 처리 기준, 결과, 검증, 한계를 포함한다.
- 수치가 있으면 원본 파일 기준인지, 전처리 결과 기준인지, 화면 예시 기준인지 명시한다.
- 검증하지 않은 내용은 확정처럼 쓰지 않는다.

## 실행 환경

- Python 작업이 생기면 Python 3.12와 `uv`를 기준으로 한다.
- 패키지 추가는 `uv add`를 사용한다.
- 테스트가 생기면 `uv run pytest`를 우선 사용한다.
