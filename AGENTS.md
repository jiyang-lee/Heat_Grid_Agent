# HeatGrid 작업 규칙

## 프론트엔드 시나리오 반영

- 홈, 알림, AI 조치, 설정 페이지를 수정할 때는 정상 시나리오와 고장 시나리오를 항상 함께 반영한다.
- 한 시나리오에서만 동작하거나 보이는 변경은 완료로 보지 않는다. 두 시나리오의 진입 경로와 화면 상태를 모두 확인한다.
- 관련 없는 화면, 컴포넌트, 스타일, 데이터는 사용자가 요청하지 않는 한 건드리지 않는다.

## 백엔드 수정 금지

- 이 저장소에서 프론트엔드 작업을 수행할 때 백엔드는 절대 수정하지 않는다.
- `simulator/versions/v2_postgres_react_ops/backend/`, `src/heatgrid_ops/`, `docker-compose*.yml`, `Dockerfile.backend`, `migrations/`, 루트 백엔드 설정은 수정 대상에서 제외한다.
- 백엔드 문제나 API 오류가 발견되어도 원인만 보고하고, 백엔드 코드는 수정하지 않는다. 백엔드 수정은 사용자가 별도로 명시적으로 요청한 경우에만 가능하다.

## 변경 범위

- 사용자가 말한 페이지와 기능만 수정한다.
- 기존 프론트엔드 API 계약과 정상/고장 시나리오 분기 구조를 우선 보존한다.
- 변경 후에는 가능한 범위에서 프론트엔드 타입 검사, 린트, 빌드와 두 시나리오의 화면 동작을 확인한다.
