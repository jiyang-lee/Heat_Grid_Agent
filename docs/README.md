# Docs Index

이 폴더는 프로젝트 문서를 목적별로 나눠 관리한다.

## 폴더 구성

- `guide/`
  - 프로젝트 규칙, 작성 기준, 고정 가이드
- `plan/`
  - 실행 계획, 설계 초안, 작업 분해
- `contracts/`
  - 구현 전에 먼저 고정하는 기술 계약서
  - state, tool 책임, I/O shape, graph 규칙
- `todo/`
  - 날짜별 작업 기록, 진행 상태 관리
- `report/`
  - 결과 보고, 정리, 회고
  - 단, 원격 업로드는 팀 공통 이해에 필요한 문서만 대상
- `send/`
  - ML팀이나 팀원에게 전달할 요청문, 공문, 안내문

## 문서 사용 원칙

- 규칙성 문서는 `guide/`에 둔다.
- 진행 중인 계획은 `plan/`에 둔다.
- 구현 기준으로 잠글 계약은 `contracts/`에 둔다.
- 매일 누적되는 작업 기록은 `todo/`에 둔다.
- 끝난 작업의 정리는 `report/`에 둔다.
- 외부 전달용 문서는 `send/`에 둔다.
- 개인 참고용 문서는 기본적으로 원격 업로드 대상이 아니다.
